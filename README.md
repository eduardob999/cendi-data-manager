# Attendance Updater

**Version 0.4 · by Eduardo Bogado (2025)**

A bulletproof CLI tool for recording daily attendance by CÉDULA number.
All data is stored in a single `nomina.csv` file that grows one column per day.

## Features

| Category | Detail |
|---|---|
| **Encoding** | Auto-detects UTF-8, UTF-8 BOM, Latin-1, CP1252, ISO-8859-1, UTF-16 |
| **Delimiter** | Auto-detects `,` `;` `TAB` `\|` |
| **Column names** | Accepts 15+ aliases for CÉDULA (`ID`, `CI`, `DNI`, `cedula`, `documento`, …) |
| **CÉDULA types** | Handles floats (`123.0`), scientific notation (`1.23e7`), spaces |
| **Date input** | 20+ formats accepted, ambiguous dates confirmed before use |
| **Blank/ragged rows** | Silently dropped on load |
| **Duplicate CÉDULAs** | Detected and warned at load time |
| **Atomic save** | Writes to a temp file first — no half-written data on crash |
| **Auto-backup** | `nomina.csv.bak` created before every save |
| **Interactive help** | Type `help` at any prompt for a full in-app Help Center |

## Building a standalone executable

`build.py` automates the full build pipeline:

```
python build.py [--clean] [--no-install]
```

### Options

| Flag | Effect |
|---|---|
| *(none)* | Install deps, then build |
| `--clean` | Delete `build/` and `dist/` before building |
| `--no-install` | Skip `pip install` (assume deps already present) |
| `--help` | Show usage |

### What the script does — step by step

1. **Pre-flight check** — verifies Python ≥ 3.8 and that `attendance.py` exists.
2. **Clean** *(optional)* — removes `build/`, `dist/`, and the `.spec` file.
3. **Install required deps** — `pandas`, `prompt_toolkit`, `colorama`, `pyinstaller`.
4. **Install optional deps** — `chardet` (skipped gracefully if unavailable).
5. **Write `requirements.txt`** — lists all deps for reproducibility.
6. **Run PyInstaller** — produces a single self-contained binary with all
   hidden imports for `pandas` and `prompt_toolkit` included automatically.
7. **Verify output** — confirms the executable was created and reports its size.
8. **Assemble package** — copies the executable and a `nomina.csv` template
   into `dist/attendance_package/` ready to hand off to the user.

### Output

```
dist/
├── attendance          ← executable (Linux / macOS)
├── attendance.exe      ← executable (Windows)
└── attendance_package/
    ├── attendance[.exe]
    └── nomina.csv
```

### Full example

```sh
# First-time build on a clean machine
python build.py --clean

# Rebuild quickly without reinstalling deps
python build.py --clean --no-install
```

## Deploying to end-users

Hand the user **two files** from `dist/attendance_package/`:

```
attendance_package/
├── attendance.exe      (or just `attendance` on Linux/macOS)
└── nomina.csv
```

> If the user already has their own `nomina.csv`, **do not overwrite it** —
> just provide the executable.

### Running the executable

| Platform | How to run |
|---|---|
| **Windows** | Double-click `attendance.exe` — a terminal window opens automatically |
| **macOS** | Open Terminal, `cd` to the folder, run `./attendance` |
| **Linux** | Open a terminal, `cd` to the folder, run `./attendance` |

> **macOS Gatekeeper:** On first run macOS may block the app.
> Right-click → Open → Open to allow it. This only needs to be done once.

> **Linux permissions:** If the file is not executable, run
> `chmod +x attendance` once.

The executable is entirely self-contained — no Python installation required
on the end-user's machine.

## nomina.csv format

The file must be in the same directory as the executable (or script).

### Minimum required column

| Column | Type | Notes |
|---|---|---|
| `CÉDULA` | Integer | Primary key — must be unique per person |

### Recommended columns

| Column | Type | Notes |
|---|---|---|
| `CÉDULA` | Integer | Numeric ID |
| `APELLIDOS` | Text | Last name(s) — auto-created blank if missing |
| `NOMBRES` | Text | First name(s) — auto-created blank if missing |

### Attendance columns (added automatically)

Each date you record creates a new column:

| Column | Type | Meaning |
|---|---|---|
| `2025-07-04` | Integer | `0` = absent, `1` = present, `2+` = multiple sessions |

### Example

```
CÉDULA,APELLIDOS,NOMBRES,2025-06-01,2025-06-02
10234567,García,María,1,0
20345678,López,Juan,1,1
30456789,Reyes,Carmen,0,1
```

### Accepted column name variants

The script recognises all of the following as the CÉDULA column:

```
CÉDULA  cédula  cedula  CEDULA  Cedula
ID  id  CI  ci  DNI  dni
documento  cedula_identidad  numero  no
```

### Accepted delimiters

`,`  `;`  `TAB`  `|` — detected automatically, no configuration needed.

### Accepted encodings

UTF-8, UTF-8 BOM (Excel default), Latin-1, CP1252, ISO-8859-1, CP1250, UTF-16
— detected automatically. Always save output as **UTF-8** when editing manually.

## Usage guide

### On startup

The program asks for a date. Any of the [formats below](#date-formats-accepted)
are accepted. It then shows a quick-reference card and opens the attendance prompt.

### At the `CÉDULA ›` prompt

| Input | Action |
|---|---|
| A CÉDULA number | Marks that person present (+1) |
| `done` | Saves `nomina.csv` and exits |
| `undo` | Reverts the last entry (stackable) |
| `add` | Registers a new person interactively |
| `help` | Opens the full in-app Help Center |
| `help dates` | Lists all accepted date formats |
| `Ctrl+C` | Aborts — **no changes are saved** |

### Auto-complete

Start typing a CÉDULA and press **TAB** to see matching suggestions from the
loaded file. New people added with `add` are immediately available in autocomplete.

### Saving

- Type `done` → `nomina.csv` is saved atomically (via temp-file rename).
- A backup `nomina.csv.bak` is written before every save.
- `Ctrl+C` at any point **discards** all changes for the current session.

## Date formats accepted

The following formats are all valid at the date prompt:

| Example | Format |
|---|---|
| `2025-07-04` | ISO 8601 *(preferred)* |
| `2025/07/04` | Slash-separated ISO |
| `2025.07.04` | Dot-separated ISO |
| `04-07-2025` | Day-Month-Year (European) |
| `04/07/2025` | Day-Month-Year (European) |
| `04.07.2025` | Day-Month-Year (European) |
| `07-04-2025` | Month-Day-Year (US) |
| `07/04/2025` | Month-Day-Year (US) |
| `4/7/2025` | Short day/month, full year |
| `4-7-25` | Short day-month, 2-digit year |
| `20250704` | Compact `YYYYMMDD` |
| `04 Jul 2025` | Day MonthAbbr Year |
| `4 July 2025` | Day MonthFull Year |
| `July 4, 2025` | MonthFull Day, Year |
| `Jul 4, 2025` | MonthAbbr Day, Year |
| `4 jul 2025` | Lowercase month name |
| `04-JUL-2025` | Uppercase month abbreviation |
| `today` | Today's date (English) |
| `hoy` | Today's date (Spanish) |

> Ambiguous dates (e.g. `04/07/2025` could be April 7 or July 4) are treated as
> **DD/MM/YYYY**. The program shows the resolved date and asks for confirmation.

## Troubleshooting

### "File not found"
`nomina.csv` must be in the same directory as the executable or script.

### "Could not find a CÉDULA column"
The column header was not recognised. Check accepted name variants above,
or open the file in a text editor and rename the column to `CÉDULA`.

### Suggestions show `12345678.0` (floats)
This was a known bug — fixed in v0.4. The CÉDULA column is now always cast
to integer automatically. If you still see it, regenerate your `nomina.csv`
from scratch or remove decimal points manually.

### File won't save — "Permission denied"
Close `nomina.csv` in Excel or any other program, then type `done` again.

### Wrong characters in names (e.g. `GarcÃ­a` instead of `García`)
The file encoding was misread. Open the file in a text editor, save it as
**UTF-8**, then re-run the program.

### macOS blocks the executable on first run
Right-click the file → **Open** → **Open**. This only needs to be done once.

### Linux — "Permission denied" when running the executable
```sh
chmod +x attendance
./attendance
```
